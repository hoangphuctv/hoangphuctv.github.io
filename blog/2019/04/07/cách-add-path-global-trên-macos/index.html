Cách add PATH global trên MacOS

Apr 7, 2019 · 2 minute read
MacOSPATHsubl

PATH là một biến môi trường trong hệ thống *nix nói cho shell biết những thư mục nào sẽ được tìm kiếm các file thực thi.

Mỗi một thư mục được các nhau bởi dấu : Đối với windows là kí tự ; Ví dụ trên MAC sau khi bạn cài sublime text thì bạn chưa thể mở file trên terminal bằng lệnh subl filename.txt được.

$ subl
bash: subl: command not found

Vậy để giải quyết được lệnh command not found thì bạn chỉ cần kiểm tra xem thư mục chứa file thực thi bạn mong muốn có nằm trong $PATH hay không. Bằng lệnh

$ echo $PATH;
/usr/local/bin:/usr/bin:/bin:/usr/sbin:/sbin

Vậy bạn thấy rằng không có thư mục nào chưa file subl cả. Vậy bây giờ ta cần làm 1 việc gọi là add PATH.

Cách add PATH như sau: Cách làm thông thường trên các hệ thống *nix Mở file ~/.bashrc Thêm dòng lệnh vào cuối file

export PATH=$PATH:/my_bin/

Nếu bạn muốn thêm cho toàn bộ user đều thấy PATH này thì thêm vào file /etc/bashrc hoặc /etc/bash.bashrc

Tên file tuỳ vào loại máy _Unix mà bạn đang xài. Nhưng tên nó sẽ có dạng /etc/_bashrc

Trên macOS bạn có thể thực hiện đơn giản hơn như sau: Bạn tìm và mở file /etc/paths. Nếu bạn có thể sử dụng vim thì gõ lệnh như sau:

sudo vim /etc/paths
  • Bấm Shift + G : về cuối file
  • Bấm Shift + A : về cuối dòng của file và bật chế độ INSERT
  • Gõ enter để xuống dòng
  • Paste vào đường dẫn chứa chương trình subl
  • Trên macOS chương trình subl nếu bạn cài đặt sublime trong /Application thì nó nằm tại /Applications/Sublime Text.app/Contents/SharedSupport/bin
  • Bấm ESC về chế độ COMMAND
  • Gõ tiếp :x để lưu file

Bây giờ bạn echo $PATH; thì nó vẫn chưa có thư mục vừa add vào đâu. Bạn cần mở 1 terminal mới.

$ echo $PATH
/usr/local/bin:/usr/bin:/bin:/usr/sbin:/sbin:/
Applications/Sublime Text.app/Contents/SharedSupport/bin

Bây giờ bạn đã thấy được có thư mục bạn vừa add vào. Bây giờ bạn đã có thể gõ lệnh

$ subl filename.text

Và sublime text đã mở ra, chứng tỏ chúng ta đã add PATH thành công.
