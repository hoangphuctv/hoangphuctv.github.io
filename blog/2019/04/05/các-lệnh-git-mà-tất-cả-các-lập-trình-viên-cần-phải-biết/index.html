Các lệnh git mà tất cả các lập trình viên cần PHẢI biết

Apr 5, 2019 · 3 minute read
git

Ngày nay đối với lập trình viên, dù là sinh viên hay lập trình viên chuyên nghiệp, khi làm việc với số lượng lớn files và projects thì bạn không thể không dùng để công cụ quản lí source code. Mà phổ biến nhất ngày xưa là SVN, và ngay nay là GIT.

Khi sử dụng git ở môi trường máy tính cá nhân, bạn hoàn toàn có thể sử dụng các phần mềm giao diện đồ hoạ như SourceTree, Github Desktop,… hoặc đơn giản nhất cũng là Git GUI.

Nhưng khi bạn làm việc trên server, đặc biệt là server Linux. Thì bạn không cài các công cụ nặng nề trên vào server mà chủ yếu sẽ sử dụng git thông qua giao diện dòng lệnh CLI. Vậy thì vừa để luyện tập vừa để HIỂU VÀ NHỚ lâu hơn. Tại sao bạn không dùng luôn các lệnh này trên máy tính của mình

git init

Khi muốn tạo project mới, bạn cd vào thư mục cần tạo. Gõ lệnh:

$ cd ~/projects/myapp
$ git init

Khi gõ lệnh git init, git sẽ tạo local repo trên thư mục đang đứng. Bây giờ bạn đã có thể tạo gói commit đầu tiên.

git clone

Khi bạn cần lấy source ở 1 link repo nào đó. Bạn gõ lệnh:

$ git clone git@github.com:laravel/laravel.git

Git sẽ tiến hành clone source code từ url tạo ra thư mục mới với tên là tên project. Ví dụ url là git@github.com:laravel/laravel.git. Thì git sẽ lấy source code về, tạo project tại thư mục laravel.

git status

Để xem tình trạng các file, file đã thay đổi; file đã xoá, file đã thêm. Ta gõ git status.

$ git status
On branch master

No commits yet

Changes to be committed:
  (use "git rm --cached <file>..." to unstage)

    new file:   readme.md

git commit

Tạo gói commit mới

Kết hợp với lệnh git status và git add.

  • git status giúp ta list ra cách file đang thay đổi (thêm, sửa, xoá)
  • git add <file> thêm file vào stagging để chuẩn bị commit.

  • git commit -m "ten commit"

    $ git status On branch master

    No commits yet

    Changes to be committed: (use “git rm –cached …” to unstage)

    new file:   readme.md

    $ git add readme.md $ git commit -m “add file readme.md”

git push

Khi làm việc với remote repo. Sau khi tạo các gói commit. Để những thành viên khác trong team có thể thấy được commit của bạn, thì bạn phải push lên remote repo.

git push

Ví dụ:

$ git push origin login-feature

git pull

Khi các lập trình viên khác, thực hiện push các thay đổi lên remote repo. Bạn lấy về bằng lệnh.

git pull

Nếu bạn may mắn code không bị conflict thì code sẽ được lấy về và merge vào nhánh hiện tại.

git fetch && git merge

Muốn lấy code từ remote về, thay vì gõ 1 lệnh git pull thì bạn có thể gõ lần lượt 2 lệnh.

git fetch

git merge remote-name/branch-name

merge branch-name trên remote với nhánh đang đứng.

Ví dụ:

$ git fetch origin master
$ git merge origin/master
